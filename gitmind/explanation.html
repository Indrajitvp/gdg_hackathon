GitMind (Discord Edition)

A zero-friction, highly scalable spaced repetition memory engine built directly into Discord.

1. The Core Concept

The Problem: Students forget 70% of what they learn within days. Mathematical "Spaced Repetition" systems (like Anki) fix this, but they require massive manual effort to create flashcards. The friction is too high.

Our Solution: Zero friction. The user pastes a wall of text into a Discord channel. Our AI automatically extracts the "Atomic Facts", saves them to a database, and the Discord bot pings the user exactly on the day they are mathematically predicted to forget it.

2. The User Journey

Phase 1: Knowledge Ingestion

1
Dump Data in Discord

The user pastes an article or sends a message directly to the Discord Bot.

2
AI Fact Extraction

The Elixir backend routes the text to the Google Gemini AI, forcing it to return a JSON array of "Atomic Facts".

3
Database Storage

Each fact is saved to Supabase PostgreSQL with a default `next_review_at` timestamp.

Phase 2: Mathematical Review

4
Daily Ping

The bot DMs the user facts they are scheduled to forget today, featuring interactive buttons: [Forgot], [Hard], [Easy].

5
Local SM-2 Math

When a button is clicked, our Elixir SuperMemo-2 engine locally calculates the exact new review date.

6
Instant Update

The database is updated in milliseconds, and the Discord message transforms to confirm the new interval.

3. The Tech Stack (Under The Hood)

Backend: Elixir & Erlang VM

We chose Elixir for its incredible concurrency. By using the `WebSockex` library, our server maintains a continuous, persistent WebSocket connection directly to the Discord Gateway. This means we process incoming messages in real-time without polling or relying on HTTP webhooks.

Database: Supabase (PostgreSQL)

We use the `Ecto` wrapper in Elixir to safely interact with a Supabase Postgres database. We maintain strict schemas for `Users` and their `Cards` (facts), ensuring we can effortlessly query millions of rows to find cards due for review on any given day.

AI: Deterministic Gemini JSON

We do NOT rely on LLMs to calculate math (they are terrible at it). Gemini is strictly used as an NLP extraction tool. We use the `application/json` response schema to force Gemini to slice raw conversational text into strict JSON strings, making our pipeline 100% crash-proof.

Algorithm: Local SuperMemo-2

The mathematical forgetting curve is calculated entirely locally in Elixir via our custom `ReviewEngine`. By keeping the SM-2 math inside our backend rather than relying on external APIs, reviews resolve in under 50 milliseconds.
